# 운영 방법

## 자동 갱신

- GitHub Actions가 매일 한국시간 09:30과 20:00에 실행됩니다.
- 09:30 배치는 KRX가 제공한 최근 거래일 종가·시가총액을 다시 확인합니다.
- 20:00 배치는 당일 상장과 장 마감 후 공시를 반영합니다.
- 기존 편입 종목은 저장된 DART 접수번호와 파싱 결과를 재사용합니다. 새 접수번호가 생긴 종목과 새로 수동편입한 종목만 문서를 파싱합니다.
- 시장데이터 갱신, 금융위 반환 확인, IPO일정 갱신은 기존과 동일하게 동작합니다.

## 첫 마이그레이션 배치

새 관리 시트가 아직 없어도 배치를 그대로 실행하면 됩니다.

1. 기존 `IPO종목`, `작업목록`, `수기입력`, `IPO일정`, `IPO기관`, `기존주주`의 값을 먼저 수거합니다.
2. 커밋된 `ipo_targets.json`, `ipo_schedule.json`, `manual_events.json`, `lockup_admin.csv`와 병합합니다.
3. `종목관리`, `보정작업`과 읽기 전용 현황 탭을 생성합니다.
4. 이후 배치부터는 새 관리 탭만 입력 원천으로 사용합니다.

행이 빠졌다는 이유만으로 삭제하지 않으며, 기존 종목을 전체 재파싱하지 않습니다. `reset_sheet` 또는 `reparse_existing`을 켤 필요가 없습니다.

## 운영자가 수정하는 탭

### 1. 종목관리

어떤 기업을 관리하고 홈페이지에 노출할지를 정하는 유일한 종목 등록 창구입니다.

| 컬럼 | 의미 |
|---|---|
| 대상구분 | `IPO일정`, `락업`, `IPO일정+락업` |
| 기업명 | DART 회사명. 신규 수동편입은 이 값만 있어도 됩니다. |
| DART기업코드 | 알면 입력합니다. 상장 전 종목 식별이 더 정확해집니다. |
| 종목코드 | 상장 전에는 공란 가능. KRX 확인 후 자동 보충됩니다. |
| 관리상태 | `자동`, `수동편입`, `검토대기`, `제외고정` |
| 노출 | 자동 종목은 공란(자동 판정), 수동편입은 `노출`, 강제 숨김은 `비공개` |
| 수동공모가 | DART 공모가를 확인할 수 없는 락업 종목만 입력 |
| 콘텐츠링크 | IPO 일정 카드의 분석 콘텐츠 링크 |

#### 이름만 먼저 추가하는 방법

`대상구분=IPO일정`, `기업명`, `관리상태=수동편입`, `노출=노출`만 입력합니다.

- 홈페이지에는 `공모 준비` 상태로 먼저 표시됩니다.
- 아직 없는 희망가·일정·주관사·공모주식수는 `미정`으로 표시됩니다.
- 배치가 매번 DART를 검색하고 파싱되는 항목부터 자동으로 채웁니다.
- DART 기업코드를 알고 있다면 같이 입력하는 것을 권장합니다.

#### 삭제 방법

행을 지우지 말고 `관리상태=제외고정`, `노출=비공개`를 선택합니다.

- 새 공시가 나와도 자동 부활하지 않습니다.
- 기존 파싱 결과는 보존되므로 다시 `자동` 또는 `수동편입`으로 바꾸면 전체 재파싱 없이 복구할 수 있습니다.
- 새 공시가 확인되어도 자동 노출하지 않고 보정작업에서 재검토할 수 있습니다.

### 2. 보정작업

IPO일정의 누락값, 락업 누락 이벤트, 기존 수기 보정과 검토 대상을 한곳에서 처리합니다.

| 컬럼 | 입력 방법 |
|---|---|
| 대상 | `IPO일정` 또는 `락업` |
| 항목 | 확정공모가·수요예측일·청약일·상장일·주관사·공모주식수·공모가·락업이벤트 등 |
| 자동값 | DART/KRX/API가 확인한 값. 직접 수정하지 않습니다. |
| 수기값 | IPO일정 보정값 또는 공모가 입력 |
| 구분/기간/수기일자/수기수량 | 락업 이벤트를 보충할 때 입력 |
| 보정방식 | `임시` 또는 `고정` |
| 처리상태 | `대기`, `적용`, `검토필요`, `자동해결`, `취소` |

- `임시`: DART 공식 값이 확인되면 자동값으로 교체되고 `자동해결` 처리됩니다.
- `고정`: DART 값이 달라도 수기값을 유지합니다.
- 빈 수기값은 자동값을 잠그지 않습니다.
- 행 삭제는 취소로 해석하지 않습니다. 사용하지 않을 작업은 `처리상태=취소`로 표시합니다.

### 3. 휴장일

거래소 휴장일을 관리합니다. 기존 형식을 그대로 사용합니다.

## 읽기 전용 탭

- `IPO일정_현황`: DART와 수기 보정을 합친 홈페이지 최종 IPO 일정
- `IPO기관_현황`: IPO기관 락업 결과
- `기존주주_현황`: 기존주주 락업 결과
- `정정이력`: 홈페이지 최종값 변경, 수기 보정 적용/해제, 고정제외 등 운영 감사기록
- `변경로그`: 자동 처리 세부 로그

구형 탭은 첫 마이그레이션 데이터 보존을 위해 남겨 두지만, 새 탭 생성 이후에는 입력 원천으로 읽지 않습니다.

## 데이터 우선순위

### 락업

1. 고정 수기값
2. 금융위 API 반환값
3. DART 파싱 예정값

수기고정값과 금융위 API가 다르면 수기값을 유지하고 보정작업에 `검토필요`로 표시합니다.

### IPO일정

1. 고정 수기값
2. 최신 DART 공시값
3. 임시 수기값

KRX는 종목코드·상장일·상장주식수·종가·시가총액을 보충합니다. DART 증권발행실적보고서가 나오면 개인청약 경쟁률과 기관 확약 배정값을 보충합니다.

## 수동 실행

GitHub의 `Actions` → `Update lockup data` → `Run workflow`에서 실행합니다.

- 일반 실행: `reset_sheet=false`, `reparse_existing=false`
- 시트만 새로 만들기: `reset_sheet=true`, `reparse_existing=false`
- 기존 DART 문서를 정말 다시 읽어야 할 때만: `reparse_existing=true`

시트 초기화는 커밋된 JSON/CSV를 재사용하므로 기존 종목 전체 파싱이 발생하지 않습니다.

## GitHub Repository Secrets

저장소의 `Settings` → `Secrets and variables` → `Actions`에 아래 값을 저장합니다.

- `KRX_API_KEY`
- `DATA_GO_KR_API_KEY`
- `DART_API_KEY`
- `GOOGLE_SERVICE_ACCOUNT_JSON`
- `GOOGLE_SHEET_ID`

`.env`와 Google 서비스 계정 JSON은 GitHub에 커밋하지 않습니다.
